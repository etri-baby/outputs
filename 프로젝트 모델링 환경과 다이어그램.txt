- 사용할 모델 타입
    > 요구사항 도출
        - Use Case Diagram (UML)
            - 요구사항 도출을 위해 사용자의 시스템 사용법 모델링
            - https://creately.com/guides/use-case-diagram-tutorial/
        - Requirement Diagram (SysML)
            - 시스템 요구사항 및 테스트케이스 모델링
            - https://astah.net/manual/sysml-and-system-safety/reference/sysml_related_function/diagrams_and_diagram_elements/requirement_dgm.html
    > 구조 설계
        - Deployment Diagram (UML)
            - 하드웨어 노드들과 하드웨어 위에 탑재될 소프트웨어 컴포넌트 모델링
            - https://creately.com/guides/deployment-diagram-tutorial/
        - Component Diagram (UML)
            - 컴포넌트간의 인터페이스 모델링
            - https://creately.com/blog/software-teams/component-diagram-tutorial/
        - Class Diagram (UML)
            - 각 컴포넌트 SW를 구성하는 class들과 그들간의 관계 모델링
            - https://creately.com/blog/software-teams/class-diagram-tutorial/
    > 행위 설계
        - Activity Diagram (UML)
            - 시스템 운행 전반의 워크플로우 모델링
            - https://creately.com/guides/activity-diagram-tutorial/
        - Sequence Diagram (UML)
            - 특정 기능의 실행 흐름 모델링
            - https://creately.com/guides/sequence-diagram-tutorial/
